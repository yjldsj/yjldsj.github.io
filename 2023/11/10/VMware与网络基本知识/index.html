VMware

虚拟机快照

VMware虚拟机快照指的就是在特定时间点保存虚拟机的状态和数据。该状态包括虚拟机的电源状态（例如，通电、断电、挂起），数据包括构成虚拟机的全部文件（例如，磁盘、内存和其他设备，如虚拟网络接口卡）。

三种网络连接方式

VMWare的虚拟机有三种网络连接方式，分别是桥接（Bridged）模式、NAT 模式（网络地址转换模式）和仅主机（Host-only）模式。

桥接（Bridged）模式

在桥接模式下，使用 VMware 创建的虚拟机就像是你买了一台新主机接到了局域网的交换机或者路由器上。它可以配置IP地址、子网掩码和其它的 TCP/IP 信息，同时它也可以访问局域网内任何一台机器。

其实在网络层面，完全可以把桥接模式的虚拟机看做是与宿主机具有同等地位的两台电脑。只要将虚拟机的 IP 地址、子网掩码配置成与宿主机同一网段，那么二者就可以进行通信。

桥接模式的使用很简单，只要将虚拟机配置为桥接模式，然后将其 IP 地址和子网掩码配置到与宿主机同一网段即可正常工作。

优先使用桥接模式的场景：局域网中的 IP 地址足够，并且局域网内的其他设备需要访问虚拟机。

虚拟系统与宿主计算机以及宿主计算机所在网络的其他计算机都可以相互访问。

桥接模式就是将主机网卡与虚拟机虚拟的网卡利用虚拟网桥进行通信。在桥接的作用下，类似于把物理主机虚拟为一个交换机，所有桥接设置的虚拟机连接到这个交换机的一个接口上，物理主机也同样插在这个交换机当中，所以所有桥接下的网卡与网卡都是交换模式的，相互可以访问而不干扰。在桥接模式下，虚拟机ip地址需要与主机在同一个网段，如果需要联网，则网关与DNS需要与主机网卡一致。其网络结构如下图所示：

NAT（网络地址转换）模式

NAT 模式就是让虚拟机借助 NAT （网络地址转换）功能，通过宿主机来访问外部网络。如果你希望你的虚拟机连接外部网络，这种模式最简单，虚拟系统不用做任何网络设置就可以访问外部网络。

在 NAT 模式下，虚拟机与宿主机的关系就像是孩子与监管很严格的父亲之间的关系。孩子想要和外面的人交流，需要先告诉父亲，然后由父亲转达。外面的人想和孩子交流，也需要先告诉父亲，然后由父亲转达给孩子。虚拟机想要访问网络，需要经过宿主机代劳。在外部设备看来，是宿主机提出了网络请求，所以将网络数据返回给了宿主机。宿主机收到数据之后，转发给虚拟机。

NAT 模式下，宿主机之外的网络设备是看不到虚拟机的存在的。如果外部设备（例如嵌入式 Linux 开发板等）想要访问虚拟机怎么办？这就需要配置 VMware 的“虚拟网络编辑器”配置端口映射，将宿主机的网络端口与虚拟机的网络端口进行映射。外部设备访问宿主机映射之后的端口就可以实现对虚拟机的访问。

NAT 模式下的虚拟系统的 TCP/IP 配置信息是由 VMnet8(NAT) 虚拟网络的 DHCP 服务器提供的，无法进行手工修改。如果只是要求虚拟机能够访问外部网络而不需要外部设备访问虚拟机，那用 NAT 模式比较方便。如果需要外部设备访问虚拟机，在 IP 地址足够的情况下，不如使用桥接模式方便。毕竟，配置端口映射也挺麻烦。

优先使用 NAT 模式的场景：

  • 场景1：局域网中的 IP 地址紧张，不能给虚拟机分配单独的 IP 地址。

  • 场景2：只需要虚拟机访问外部网络，不需要外部设备访问虚拟机。

虚拟机可以访问宿主计算机所在网络的其他计算机（反之不行），可以与宿主计算机互访。

如果你的网络ip资源紧缺，但是你又希望你的虚拟机能够联网，这时候NAT模式是最好的选择。NAT模式借助虚拟NAT设备和虚拟DHCP服务器，使得虚拟机可以联网。其网络结构如下图所示：

仅主机（Host-only）模式

在有些情况下，我们需要 将虚拟机所在的网络环境与宿主机所在的网络环境进行隔离，这个时候就可以采用 Host-only 模式。

在 Host-only 模式下：

  • 所有的虚拟机之间可以相互通信

  • 虚拟机与宿主机之间可以相同通信，类似于将二者通过双绞线互连。

  • 虚拟机与宿主机所在局域网中的其它设备之间不能通信。

在 Host-only 模式下，虚拟系统的 TCP/IP 配置信息(如 IP 地址、网关地址、DNS 服务器等)，都是由 VMnet1 (host-only) 虚拟网络的 DHCP 服务器来动态分配的。

这种模式可以把 VMnet1 想象成一个路由器，它为所有连接上的虚拟机分配 IP 地址。宿主机也通过 VMware Network Adapter VMnet1 虚拟网卡连接到 VMnet1 上，所以也会得到一个 IP 地址。VMnet1 管理下的所有机器（虚拟机、宿主机）之间可以正常通信。

除了 VMware Network Adapter VMnet1 虚拟网卡之外，宿主机还会有物理网卡，并通过物理网卡连接到了现实中的交换机或者路由器上。物理网卡所在的网络和 VMware Network Adapter VMnet1 虚拟网卡所在的网络之间是完全隔离的，所以物理局域网内的其他设备和 VMnet1 管理下的虚拟机之间无法通信。

优先使用 Host-only 模式的场景：

  • VMware 虚拟机与物理局域网内的其他设备需要进行网络隔离。

这种模式下，虚拟系统的网卡连接到宿主计算机的VMware Network Adapter VMnet1网卡上。虚拟系统的TCP/IP配置信息(如IP地址、网关地址、DNS服务器等)，都是由VMnet1(host-only)虚拟网络的DHCP服务器来动态分配的。

默认情况下，虚拟系统只能与宿主计算机互访，这也是Host-only的名字的意义。此时相当于两台机器通过双绞线直连。

Host-Only模式其实就是NAT模式去除了虚拟NAT设备，然后使用VMware Network Adapter VMnet1虚拟网卡连接VMnet1虚拟交换机来与虚拟机通信的，Host-Only模式将虚拟机与外网隔开，使得虚拟机成为一个独立的系统，只与主机相互通讯。

ping过程

在 VMware 下，两个虚拟机之间的通信可以通过 NAT 模式和桥接模式来实现。

NAT 模式

在 NAT 模式下，两个虚拟机之间的通信需要通过宿主机作为中介。宿主机会为每个虚拟机分配一个虚拟 IP 地址，并将虚拟 IP 地址映射到宿主机的物理 IP 地址。当一个虚拟机向另一个虚拟机发送数据时，数据会先经过宿主机的 NAT 设备，然后再发送到另一个虚拟机。

桥接模式

在桥接模式下，两个虚拟机之间可以直接通信，而不需要经过宿主机。虚拟机之间会建立一个虚拟网络，虚拟网络中的所有虚拟机都可以直接通信。

Ping 的详细过程和原理

Ping 是用来测试网络连接是否正常的一种工具。当你在一个虚拟机上执行 ping 命令时，虚拟机会向另一个虚拟机发送一个 ICMP 请求数据包。如果另一个虚拟机收到了 ICMP 请求数据包，并返回了一个 ICMP 响应数据包，那么 ping 命令就会显示成功。

在 NAT 模式下，当一个虚拟机向另一个虚拟机发送 ICMP 请求数据包时，数据包会先经过宿主机的 NAT 设备。NAT 设备会将 ICMP 请求数据包的源 IP 地址和源端口号修改为宿主机的物理 IP 地址和物理端口号，然后再将数据包发送到外网。

外网中的路由器会根据 ICMP 请求数据包的目的 IP 地址，将数据包转发到另一个虚拟机所在的网络。另一个虚拟机的网卡会收到 ICMP 请求数据包，并将数据包发送给操作系统。操作系统会将 ICMP 请求数据包转发给内核，内核会将 ICMP 请求数据包转发给 ICMP 协议栈。ICMP 协议栈会将 ICMP 请求数据包封装成一个 ICMP 响应数据包，并将数据包发送给发送端。

在桥接模式下，当一个虚拟机向另一个虚拟机发送 ICMP 请求数据包时，数据包会直接发送到另一个虚拟机。另一个虚拟机的网卡会收到 ICMP 请求数据包，并将数据包发送给操作系统。操作系统会将 ICMP 请求数据包转发给内核，内核会将 ICMP 请求数据包转发给 ICMP 协议栈。ICMP 协议栈会将 ICMP 请求数据包封装成一个 ICMP 响应数据包，并将数据包发送给发送端。

总结

在 NAT 模式下，两个虚拟机之间的通信需要通过宿主机作为中介。在桥接模式下，两个虚拟机之间可以直接通信。Ping 是用来测试网络连接是否正常的一种工具。当你在一个虚拟机上执行 ping 命令时，虚拟机会向另一个虚拟机发送一个 ICMP 请求数据包。如果另一个虚拟机收到了 ICMP 请求数据包，并返回了一个 ICMP 响应数据包，那么 ping 命令就会显示成功。

IP地址

IP地址（Internet Protocol Address）是指互联网协议地址，又译为网际协议地址。IP地址是一种统一的地址格式，它为互联网上的每一个网络和每一台主机分配一个逻辑地址，以此来屏蔽物理地址的差异。

IP地址分为IPV4和IPV6，我们所说的的IP地址指的是IPV4的地址。

IPV4互联协议版本4，有版本V4之前就有IPV1 IPV2IPV3，同样有IPV5 IPV6，IPV5在实验中已经夭折，替代IPV4的将是IPV6。(IPV4地址比如：192.168.1.1、114.114.114.114，IPV6地址比如：2001::1）

注：接下来我们讲的IP指的是IPV4地址

IP是电子设备间的名字。IP地址是由二进制组成的，但为了方便人的记忆转换为十进制。

IP地址是由两部分组成的，网络部分和主机部分，比如：192.168.0.11，192.168.0是网络部分，11是主机部分。

网络部分如果一样代表在同一个网段（主机部分可以不一样）。

什么叫同一个网段（同一个广播域）呢？好比在同一房间的人一样，他们之间通讯可以基本靠吼，也就是我们所说的广播。不同网段的好比不同房间的他们之间正常情况下不能通讯。

IP地址分为4组，8bit（8个二进制）一组，4个组组成了32个二进制。二进制1111 1111转换为十进制为255 。这个32位的数根据需要被分为五大类分，由IANA(全球互联网地址指派机构)配给大家使用。

IP地址类型

IP地址类型分为A、B、C、D、E。

net-id是网络部分（NETWORK），host-id是主机部分（HOST）。

A、B、C三类中 IP地址 = 网络地址 + 主机地址 ，而D、E两类不区分网络地址和主机地址

一般来说这五类的用途大致分为：

  • A类
    • 保留给政府机构（ 0.0.0.0 到 127.255.255.255 ）
    • 默认子网掩码：255.0.0.0
    • 私有号段/私网地址：10.0.0.0 到 10.255.255.255
  • B类
    • 分配给中等规模的公司（ 128.0.0.0 到 191.255.255.255 ）
    • 默认子网掩码：255.255.0.0
    • 私有号段/私网地址：172.16.0.0 到 172.31.255.255
  • C类
    • 分配给任何需要的人（ 192.0.0.0 到 223.255.255.255 ）
    • 子网掩码：255.255.255.0
    • 私有号段/私网地址：192.168.0.0 到 192.168.255.255
  • D类
    • 用于组播（ 224.0.0.0 到 239.255.255.255 ）
  • E类
    • 用于将来和实验（ 240.0.0.0 到 247.255.255.255 ）

特殊说明：

  • A类中的其中127和0开头的为特殊地址。127.X.X.X 是保留地址。
    • 例如 127.0.0.1 等效于 localhost 或 本机IP。 一般用于测试使用。 ping 127.0.0.1 来测试本机TCP/IP是否正常。
  • 0.0.0.0。地址类型：所有网络。用途：华为VRP路由器，用于指定默认路由。
  • 255.255.255.255。地址类型：广播地址。用途：本网段所有节点。
  • 公网地址：出了私网地址以外的地址；公网使用
  • 私网地址：
    • 10.0.0.0 到 10.255.255.255
    • 172.16.0.0 到 172.31.255.255
    • 192.168.0.0 到 192.168.255.255
    • 主要给公司内网使用
  • 主机部分全为 0 的。地址类型：网络地址。用途：代表一个网段。
  • 主机部分全为 1 的。地址类型：广播地址。用途：特定网段的所有节点。

子网掩码

“子网”掩码，顾名思义，它就是拿来划分子网的，更准确的说，划分子网的同时，还能通过它知道主机在子网里面的具体ip的具体地址。

用途

那么划分这个子网有什么用呢？

  1. 方便管理，不同的子网可以相互隔离
  2. 我们知道，IPv4的ip地址资源比较紧张，实际上整个互联网就是一个巨大的局域网，ip地址有限，那么通过划分子网，如何再在子网里面分配ip地址给子网的主机，这个时候同一一个ip就可以给一整个子网所共用，就解决了互联网ip不够的问题

原理

  • 既然要划分子网，要有一个规则，告诉路由器，IP的哪一部分是表示子网，哪一部分表示我在这个子网里面具体的地址。
  • 所以，实际上IP细来讲又分为三部分：网络部分（network）、子网段部分（subnetwork）和主机部分（host）。
  • 一般大家喜欢把 网络部分和子网部分 合并成为 网络地址（网络号），把子网看做是网络的一部分，主机部分称为 主机地址（主机号）。

那么，是怎么样计算出来网络号和主机号的呢？

子网掩码会把一个IP地址按这个划分成 网络号 和 主机号 两部分，计算方法是通过把IP地址变成二进制，子网掩码也转成二进制后，两个二进制数按位做 与运算。

比如，一个C类地址，ip: 192.168.0.3 ，子网掩码是：255.255.255.0。将其二进制按位做与计算。得到结果192.168.0.0。如下所示：

192.168.0.0 这个就是我们的 网络段 （包括了上面说的网络部分和子网部分）。3 就是我们的主机号（主机在子网络里面的标识）。

上面这个算法看起来有点难理解，另外一种 CIDR(无类别域间路由)/VLSM（可变长子网掩码） 的表示法跟容易看懂，如下面这个：192.168.0.3/24 。

24 表示这个IP的 32 位二进制数中的前 24 位是 网络标识的前缀，剩下的8位是 主机标识。

严格来，剩下的8位里面，全部是0是这个网络的标识，全部是1是这个网络的子网内广播地址，其他的才是这个子网网络的可用地址 。

实际上，这个 192.168.0.3/24 ，前24位全1，就是等于 255.255.255.0，这个就是这个网络的子网掩码 。

而无类别域间路由是基于可变长子网掩码（VLSM）来进行任意长度的前缀的分配的，所以这个24也可以变成其他数，如16，192.168.0.3/16 表示只有前16位是网络标识，换算回来就是 192.168.0.0 是网络标识（后16位二进制全位0）， 192.168.255.255 这个是域内广播地址（后16位二进制全为1）。

那么 192.168.0.0–192.168.255.255 间所有的其他地址就是这个子网的可以容纳的主机地址。所以这个子网的地址数量明显比192.168.0.3/24那个要大多了。

那么的子网掩码也很容易计算，就是前面16位全是1后面16位全是0，换算成10进制就是 255.255.0.0 。

注意：

  • 网络地址：网络地址是指一个网络中的第一个地址，用于标识整个网络。对于给定的子网，网络地址表示该子网的起始地址，通常是该子网中所有设备的共同网络标识符。

  • 广播地址：广播地址是指一个网络中的最后一个地址，用于向该网络中的所有设备发送广播消息。当某个设备发送广播消息时，这条消息将会被发送到该网络上的所有设备，并且只有该网络上的设备会接收到这条消息。广播地址通常是该子网中所有设备的共同广播目标。

在 IPv4 的网络中，网络地址和广播地址各自具有特殊的含义和用途。因此，在计算可用的 IP 地址数量时，需要排除掉网络地址和广播地址，因为它们分别代表了整个网络和广播消息的目标，而不是可以分配给单独设备的实际可用 IP 地址。

网关

1、网关用于实现两个不同网段的设备(IP)进行通信。

2、具有路由功能的设备(路由器)的IP地址就是网关的IP地址。

网关（Gateway）： 就是一个网络连接到另一个网络的“关口”，实质上是一个网络通向其他网络的IP地址。

网关作用： 实现两个不同网段的设备(IP)进行通信。

  • 有网络A：IP地址范围为 192.168.1.1~192.168.1.254 ，子网掩码为 255.255.255.0 。
  • 有网络B：IP地址范围为 192.168.2.1~192.168.2.254 ，子网掩码为 255.255.255.0 。

在没有路由器的情况下，两个网络之间是不能进行TCP/IP通信的，即使是两个网络连接在同一台交换机（或集线器）上，TCP/IP协议也会根据子网掩码（255.255.255.0）判定两个网络中的主机处在不同的网络里。而要实现这两个网络之间的通信，则必须通过网关。

举例：

在上面这种情况下，假设网络A和网络B是两个独立的局域网（LAN），它们使用不同的IP地址范围和子网掩码。如果希望这两个网络之间进行通信，需要一个连接这两个网络的设备，通常是路由器。

  1. 设置网关：
    • 对于网络A中的设备，其默认网关应该是属于网络A的一个路由器接口，比如 192.168.1.1 。
    • 对于网络B中的设备，其默认网关应该是属于网络B的一个路由器接口，比如 192.168.2.1 。
  2. 路由器设置：
    • 为了让网络A和网络B之间进行通信，你需要在路由器上配置两个接口，分别连接到网络A和网络B，并分别分配对应的IP地址，比如192.168.1.1和192.168.2.1。
  3. 数据包转发：
    • 当网络A中的设备需要与网络B中的设备通信时，数据包首先发送到网络A的默认网关（即路由器）。
    • 路由器收到数据包后，根据目标IP地址判断应该将数据包发送到网络B中的哪个设备，然后进行相应的转发。

总之，在这种情况下，要实现网络A和网络B之间的通信，你需要一个能够连接这两个网络的路由器，并且在路由器上进行相应的配置以确保数据包能够在这两个网络之间正确地转发。

默认网关

默认网关就是网络设备（通常是路由器）的IP地址，用于将数据包从本地网络发送到其他网络。

通常情况下，网关的IP地址是网络中第一个可用的IP地址，即与子网的网络地址相同，只是最后一位不同。但是，这并不是绝对的规则，有些网络可能会使用不同的IP地址作为网关。在实际网络中，网关的IP地址是由网络管理员进行配置的，因此可能会根据实际需要进行设置。

请注意，默认网关的设置对于计算机的网络通信至关重要。如果默认网关配置不正确，计算机可能无法与其他网络通信，因此确保正确配置默认网关非常重要。

默认网关的最后一部分通常是由网络管理员或路由器配置决定的：

  1. 网络管理员：
    • 在一个局域网（LAN）中，网络管理员负责配置网络设备，包括路由器和交换机。他们会为每个设备分配IP地址，其中包括默认网关。
    • 默认网关就是网络设备（通常是路由器）的IP地址，因为路由器是连接本地网络和其他网络的关键设备。网络管理员会根据网络拓扑和需求来选择默认网关的IP地址。
  2. 路由器配置：
    • 路由器是将数据包从一个网络传递到另一个网络的设备。它具有多个接口，每个接口都与不同的网络相连。
    • 路由器的配置文件中包含了默认网关的IP地址。这通常是在路由器设置中手动配置的。
    • 路由器的默认网关通常是与其连接的ISP（互联网服务提供商）的路由器的IP地址。这样，路由器可以将数据包传递到互联网上的其他网络。

IP配置

IP地址配置通常有两种方式：

  1. 静态IP地址配置：管理员手动分配IP地址、子网掩码、默认网关和DNS服务器等网络参数。这些配置不会自动更改，因此设备始终使用相同的IP地址。
  2. 动态IP地址配置：通过 DHCP（动态主机配置协议） 自动分配IP地址、子网掩码、默认网关和DNS服务器等网络参数。设备在连接到网络时会自动获取这些信息。（并且可能会在每次连接时获得不同的IP地址）

如果一个局域网内，有一个动态分配的IP192.168.2.4，有一个手工配置的静态IP192.168.2.4。

可能出现的情况不一定（取决于路由规则，和具体情况判定）

  • 是先动态分配的，然后是手动配置的，可能手动配置的上不了网
  • 也可能两个都上不了网

三层结构模型

三层结构模型是指网络架构中常见的设计，通常用于大型企业或组织的网络环境。这个模型包括接入层、汇聚层和核心层，每一层都有其独特的功能和作用。

接入层

  • 定义：接入层是网络中最靠近用户的一层，它连接着终端设备（如计算机、手机、打印机等）到局域网（LAN）或广域网（WAN）。
  • 功能：在接入层，主要的功能是提供对用户的接入，并负责将用户数据传输到汇聚层。它还承担了安全性、QoS（服务质量）和 VLAN（虚拟局域网）的实现。
  • 特点：接入层通常需要考虑到大量的用户连接，因此需要具备较高的可靠性和灵活性。同时，它也需要能够处理不同种类的数据流量，包括语音、视频和数据。

汇聚层

  • 定义：汇聚层位于接入层和核心层之间，主要负责连接多个接入层交换机，并将来自接入层的流量进行汇总和转发。
  • 功能：在汇聚层，主要的功能是实现不同接入层的数据聚合，并提供对核心层的连接。它也负责实施路由、过滤和策略控制。
  • 特点：汇聚层需要具备较高的带宽和处理能力，以应对来自多个接入层的数据流量。同时，它也需要能够支持多种协议和服务，以确保数据的快速和可靠传输。

核心层

  • 定义：核心层是网络的中枢部分，主要负责连接各个汇聚层，并处理大量的数据传输。
  • 功能：在核心层，主要的功能是实现高速的数据交换和转发。它需要支持高容量的数据传输，并提供高度可靠的连接。
  • 特点：核心层通常需要具备非常高的带宽和处理能力，以应对大规模的数据交换。同时，它也需要具备高度的可靠性和容错能力，以确保整个网络的稳定运行。

这个三层结构模型通过明确定义每一层的功能和特点，可以帮助设计者更好地规划和实现复杂网络环境，从而确保网络的高效性和稳定性。

OSI七层模型

OSI七层模型

OSI七层模型是一种用于理解和描述计算机网络体系结构的标准框架。以下是每个层次的详细解释：

  1. 应用层（Application Layer）：应用层为用户提供网络服务，包括文件传输、电子邮件、远程登录等各种应用。它与用户接口和应用程序之间的通信有关。
  2. 表示层（Presentation Layer）：表示层负责数据的格式转换、加密解密、压缩解压缩等功能，使不同系统能够互相理解数据。它还处理数据的语法和语义问题。
  3. 会话层（Session Layer）：会话层负责建立、管理和终止会话连接，以及数据交换的顺序和同步。它还处理数据的传输方式，如全双工或半双工。
  4. 传输层（Transport Layer）：传输层负责端到端的通信连接，提供可靠的数据传输服务。它通过分段和重组数据来确保数据的完整性和可靠性，并进行流量控制和错误检测。
  5. 网络层（Network Layer）：网络层负责在不同网络之间传输数据包，并处理寻址和路由选择。它的主要功能是确定数据的路径，以便数据能够从源节点传输到目的节点。
  6. 数据链路层（Data Link Layer）：数据链路层负责在相邻节点之间传输数据帧，并提供错误检测和纠正功能。它分为两个子层：逻辑链路控制（LLC）和介质访问控制（MAC）。
  7. 物理层（Physical Layer）：这一层负责管理数据传输的物理介质，包括电压、光信号、无线电频率等。它定义了连接设备的接口标准、传输速率和编码技术。

OSI七层模型提供了一个清晰的分层结构，有助于不同厂商设备之间的互操作性，同时也方便网络设备和软件开发人员了解网络通信的不同方面。

OSI七层常用的协议

以下是 OSI 模型每一层常用的协议：

  1. 应用层：
    • HTTP （超文本传输协议）（HyperText Transfer Protocol）（端口号80）
      • 用于在Web浏览器和Web服务器之间传输超文本的协议。
    • HTTPS （HyperText Transfer Protocol Secure）（端口号443）
      • 是HTTP的安全版本，通过使用SSL/TLS加密来保护数据传输的安全性。
    • FTP （文件传输协议）（File Transfer Protocol）（端口号21）
      • 用于在网络上进行文件传输的标准协议。
    • SMTP （简单邮件传输协议）（Simple Mail Transfer Protocol）（端口号25）
      • 用于在计算机网络上发送电子邮件的协议。
    • DNS （域名系统）（Domain Name System）（端口号53）
      • 用于将域名解析为对应的IP地址的分布式数据库系统。
    • DHCP （动态主机配置协议）（Dynamic Host Configuration Protocol）（端口号67/68）
    • POP3 （Post Office Protocol 3）（端口号110）
      • 用于从远程服务器上获取电子邮件的协议。
    • IMAP （Internet Message Access Protocol）（端口号143）
      • 也是用于电子邮件的协议，允许用户从远程服务器上检索电子邮件。
    • SMB （Server Message Block）（端口号445）
      • 使用 TCP 协议在网络上传输数据。用于在网络上共享文件和打印机。SMB 协议是 Windows 操作系统的默认文件共享协议，但它也可以在其他操作系统上使用。
    • RDP （Remote Desktop Protocol）（端口号3389）
      • 使用 TCP 协议在网络上传输数据。用于远程访问和控制计算机。RDP 协议是 Microsoft Windows 操作系统的默认远程访问协议，但它也可以在其他操作系统上使用。
  2. 表示层：表示层也没有常用的特定协议，但它涉及数据格式、编码和加密解密等方面。
    • ASCII码，SSL和TLS提供安全通信，JPEG(联合图像专家组)和MPEG(动态图像压缩标准)用于图像和视频压缩
  3. 会话层：会话层并没有常用的特定协议，但它定义了建立、管理和终止会话连接的规则。
    • NetBIOS（网络基本输入输出系统）：用于网络通信
    • RPC（远程过程调用协议）
    • PPTP用于虚拟专用网络
    • SSH（安全外壳协议）（Secure Shell）（端口号22）
      • 用于在网络中提供加密的通信会话的协议。SSH 协议使用加密和认证机制来保护数据和通信。
      • SSH 协议可以用于远程访问计算机、传输文件和管理服务。
      • SSH 协议使用 TCP 协议作为传输层协议。它在应用层和传输层之间提供安全连接。
  4. 传输层：
    • TCP （传输控制协议）（Transmission Control Protocol）
      • 提供可靠的、面向连接的通信服务。
    • UDP （用户数据协议）（User Datagram Protocol）
      • 提供无连接的通信服务，适用于实时应用。
    • SCTP （流控制传输协议）
  5. 网络层：
    • IP （Internet协议）（Internet Protocol）
      • IP协议是网络层的核心协议，负责在网络中实现主机之间的通信和数据包的路由。
      • IP地址用于标识网络中的设备，IPv4和IPv6是两种常见的IP版本。
    • ICMP （Internet控制消息协议）（Internet Control Message Protocol）
      • ICMP是用于在IP网络上发送控制消息的协议，用于测试目的、错误报告和管理功能。
      • 它可以用于检测主机是否可达、测量往返时间（ping）以及发出错误报告。
    • ARP（Address Resolution Protocol）
      • ARP协议用于将IP地址映射到MAC地址，以便在局域网中找到目标设备。
      • 当需要与另一台设备通信时，ARP协议会根据IP地址获取对应的MAC地址。
    • RIP（Routing Information Protocol）
      • RIP是一种用于在小型网络中交换路由信息的动态路由协议，帮助路由器决定数据包的最佳路径。
      • RIP使用跳数作为路由选择的度量标准，适用于较小规模的网络，但对大型网络不够适用。
    • OSPF （开发最短路径优先协议）（Open Shortest Path First）
      • OSPF是一种基于链路状态的动态路由协议，用于在复杂网络中计算最佳路径，并支持更大规模的网络。
      • OSPF通过派发链路状态信息并计算最短路径来确定路由，适合于大型企业网络或互联网服务提供商的使用。
    • IPSec （IP安全性协议）
  6. 数据链路层：
    • 以太网（Ethernet）
    • PPP（Point-to-Point Protocol）
    • HDLC（High-Level Data Link Control）
    • Ethernet（IEEE 802.3）定义了数据帧的格式和MAC地址的使用
    • PPP（点对点协议）
    • HDLC（高级数据链路控制）
    • ARP（地址解析协议）
  7. 物理层：
    • 常见的物理层协议包括以太网、Wi-Fi、蓝牙等。
    • Ethernet、RS-232、USB、光纤等
    • RJ45用于以太网接口，IEEE 802.11用于Wi-Fi无线网络

这些协议在计算机网络中扮演着重要的角色，负责实现不同层次的功能，并确保网络通信的顺利进行。

DNS（域名系统）

DNS （域名系统）（Domain Name System）（端口号53）

DNS 域名解析是将域名转换为 IP 地址的过程。当你在浏览器中输入一个域名，例如 www.google.com，你的计算机会将该域名发送到 DNS 服务器。DNS 服务器会查找该域名的 IP 地址，并将该 IP 地址返回给你的计算机。你的计算机然后使用该 IP 地址来连接到 Google 的服务器。

DNS 域名解析过程如下：

  1. 用户在浏览器中输入一个域名，例如 www.google.com。
  2. 计算机将该域名发送到 DNS 服务器。
  3. DNS 服务器查找该域名的 IP 地址。
  4. DNS 服务器将该 IP 地址返回给计算机。
  5. 计算机使用该 IP 地址来连接到 Google 的服务器。

DNS 域名解析过程可能会经过 hosts、路由器、国家、根 DNS。

  • hosts 文件是存储在本地计算机上的文件，它包含域名和 IP 地址的映射。如果计算机的 hosts 文件中包含了某个域名的 IP 地址，那么计算机就会直接使用该 IP 地址来连接到该域名，而不会通过 DNS 服务器。
  • 路由器 是连接到互联网的设备，它负责将数据包从一个网络转发到另一个网络。路由器可以缓存 DNS 查询，这样它就可以在以后直接响应相同的查询，而不需要向 DNS 服务器发送查询。
  • 国家 有不同的 DNS 服务器，这些服务器负责管理不同的域名。当计算机向 DNS 服务器发送域名查询时，DNS 服务器会根据域名的后缀来确定应该将查询发送到哪个国家。
  • 根 DNS 是负责管理顶级域名（例如 .com、.net 和 .org）的 DNS 服务器。当计算机向 DNS 服务器发送域名查询时，DNS 服务器会首先将查询发送到根 DNS。根 DNS 会将查询转发到负责管理该域名的 DNS 服务器。

DNS 域名解析过程通常是自动完成的，用户不需要进行任何操作。

http、https、socket4、socket5的区别

这些协议在网络通信中扮演着不同的角色，以下是它们之间的区别：

HTTP (HyperText Transfer Protocol)

  • 作用：HTTP是一种用于传输超文本的协议，通常用于在Web浏览器和Web服务器之间传输数据。
  • 特点：它是无状态的，每个请求都是独立的，不会保存之前的状态信息。
  • 端口：默认端口为80。

HTTPS (HyperText Transfer Protocol Secure)

  • 作用：HTTPS是HTTP的安全版本，通过使用SSL/TLS加密来保护数据传输的安全性。
  • 特点：数据在传输过程中经过加密处理，因此更安全。
  • 端口：默认端口为443。

SOCKS4 (Socket Secure 4)

  • 作用：SOCKS4是一种网络传输协议，允许客户端将其流量转发到服务器上。
  • 特点：SOCKS4只支持TCP连接，不支持UDP，也不支持身份验证。
  • 端口：默认端口为1080。

SOCKS5 (Socket Secure 5)

  • 作用：SOCKS5是SOCKS4的升级版本，增加了对UDP、身份验证等功能的支持。
  • 特点：相对于SOCKS4，SOCKS5更加灵活，同时增加了安全性和功能性。
  • 端口：默认端口为1080。

总的来说，HTTP和HTTPS主要用于Web数据传输，而SOCKS4和SOCKS5则更多用于代理服务，提供更广泛的网络传输支持。HTTPS相比HTTP更为安全，而SOCKS5相比SOCKS4增加了更多的功能和安全性。

Socket是一种用于网络通信的编程接口，它允许计算机程序通过网络进行数据交换。在网络编程中，程序员可以利用Socket来实现客户端和服务器之间的通信。

Socket提供了一种标准的接口，使得开发人员可以使用各种编程语言（如Python、Java、C++等）来操作网络通信。通过Socket，程序可以建立连接、发送数据、接收数据以及关闭连接。它支持多种类型的通信，包括TCP（传输控制协议）和UDP（用户数据报协议），因此可以满足不同的通信需求。

在Socket编程中，通常会涉及到一些重要的概念，比如IP地址、端口号以及套接字类型等。程序员可以利用这些概念来实现网络通信，并处理各种网络通信中可能出现的问题，比如连接超时、数据丢失等。

UDP和TCP

UDP (User Datagram Protocol) 和 TCP (Transmission Control Protocol) 都是 OSI 模型中传输层的两种不同协议，它们有一些显著的区别：

UDP (User Datagram Protocol)

  • 特点：UDP是一种简单的无连接协议，它不保证数据传输的可靠性或顺序。
  • 适用场景：常用于实时应用，如音频/视频流、在线游戏等，对数据传输的实时性要求较高的场景。
  • 优点：UDP的头部开销小，传输速度快，适合于短小的数据包传输。
  • 缺点：由于其无连接性质，数据包可能会丢失或乱序。

TCP (Transmission Control Protocol)

  • 特点：TCP是一种面向连接的协议，提供可靠的、有序的数据传输服务。
  • 适用场景：常用于需要可靠数据传输的应用，如文件传输、Web浏览等。
  • 优点：TCP通过确认机制和重传来保证数据的可靠传输，能够处理数据包的丢失和乱序。
  • 缺点：相对于UDP，TCP的头部开销较大，传输速度相对较慢。

总结

  • UDP更适合于实时应用，对数据传输的时效性要求较高，但在数据可靠性方面则较为薄弱。
  • TCP适合于需要可靠数据传输的应用，尤其是需要确保数据不丢失、不重复、按顺序到达的场景。

路由器、路由表、交换机

路由器是一种网络设备，用于在不同网络之间转发数据包，并且能够连接多个计算机网络。它可以根据目的地址来决定数据包的传输路径，并负责在网络中进行数据包的转发。路由器通常使用IP地址和路由表来确定数据包的最佳传输路径。

路由表是路由器内部存储的一个表格，其中列出了路由器所知道的网络地址及其对应的下一跳地址（即数据包应该被发送到的下一个路由器）。路由表还包含有关这些网络地址的其他信息，如子网掩码、接口等。当路由器收到一个数据包时，它会查找路由表来确定最佳的传输路径，然后将数据包转发到相应的下一跳地址。

路由器的主要功能和作用包括：

  1. 网络互连：连接不同的计算机网络，实现不同网络之间的通信。
  2. 数据包转发：根据目的地址从路由表中选择最佳的传输路径，并将数据包转发到下一个路由器或目的地。
  3. 路由选择：通过动态路由协议或手动配置来更新路由表，以确保数据包按照最佳路径传输。

交换机是一种网络设备，用于在局域网（LAN）中连接多台计算机，并且能够根据目的地址来转发数据帧。它通常用于构建局域网内部的数据传输网络。

交换机的主要功能和作用包括：

  1. 数据帧转发：根据目的MAC地址将数据帧从一个端口转发到另一个端口，以实现对特定计算机之间的直接通信。
  2. 网络分段：将局域网划分成多个虚拟的子网，以提高网络性能和安全性。
  3. 广播控制：控制广播数据包的传输范围，避免网络中出现过多的广播风暴。

公网、私网、外网、内网

公网（Public Network）:

  • 公网是指全球范围内的互联网，所有人都可以访问。
  • 设备在公网中使用全局唯一的公有IP地址，这些IP地址是全球唯一的，并且可以被路由到全球任何地方。
  • 公网允许设备之间的全球性通信，可以访问互联网上的各种服务和资源。

私网（Private Network）:

  • 私网是一个更广泛的术语，可以用来描述任何非公共的网络，包括但不限于内网。
  • 它还可以指代个人家庭网络、虚拟专用网络（VPN）等。
  • 与内网不同，私网并不一定局限于单个组织或公司，它可以包括多个组织之间的连接，或者通过加密隧道将远程用户连接到公司网络中。

外网（Extranet）:

  • 外网是指两个或多个组织之间共享信息的网络，通常是通过互联网实现的。
  • 外网允许不同组织之间的安全通信和资源共享，通常需要进行身份验证和授权才能访问共享的资源。

内网（Intranet）:

  • 内网是指一个组织或公司内部的网络，用于内部通信、资源共享和应用程序访问。
  • 它通常由一组相互连接的局域网组成，可以通过专门的网络设备（如路由器、交换机）进行连接。
  • 内网中的设备使用专门的IP地址范围，通常是私有IP地址。

区别:

  1. 可访问性: 公网是全球范围内可访问的网络，而私网、外网和内网通常是受限访问的，只允许特定范围内的设备或用户进行访问。
  2. 范围: 外网是指不同组织之间共享信息的网络，而内网通常指单个组织或公司内部的网络。私网是一个更广泛的术语，可以覆盖个人家庭网络、VPN等。
  3. 连接方式: 外网通常涉及跨组织之间的连接和共享，而内网通常用于组织内部通信和资源共享。

端口号

端口号是计算机网络中用于标识通信端点的数字。端口号通常在0到65535之间。但是，有些端口号被保留用于特殊目的，例如网络服务。

以下是一些常见的保留端口号：

  • 0-1023：保留给系统服务
  • 1024-49151：可供用户分配
  • 49152-65535：保留给私有使用

如果您想使用端口号来创建自己的网络服务，您可以选择一个在1024到49151之间的端口号。但是，您应该确保该端口号没有被其他服务使用。您可以使用netstat命令来查看哪些端口正在使用。

如果您想使用端口号来连接到其他网络服务，您需要知道该服务使用的端口号。您可以通过查看该服务的文档或使用netstat命令来查找该端口号。
