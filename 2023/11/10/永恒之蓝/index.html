永恒之蓝

Posted on 2023-11-10 Edited on 2023-11-14 In 网络

前提

有 cn_windows_7_ultimate_with_sp1_x64_dvd_u_677408.iso 镜像文件，并且创建了虚拟机win7，并且开启桥接模式或NAT模式，建议开桥接模式

有 kali-linux-2023-W10-vmware-amd64.vmwarevm.zip 文件，解压并且添加到虚拟机上，开启桥接模式或NAT模式，建议开桥接模式

这次是kali攻击win7。攻击机-kali：192.168.0.65。靶机-win7：192.168.0.64

两台机子可以相互ping通，并且靶机（无补丁）开启了445端口，防火墙是关闭的！

对win7的cmd ipconfig 一下，对kali的终端 ifconfig 一下，使得二者可以相互 ping 通

如果在kali上不小心使用了 ping 命令，使用 Ctrl + C 中断。

如果不放心可以查看进程 ps，再去查查PID和怎么杀死进程。

1、主机发现：

登录 kali linux，用 nmap 探测本网段存活主机

可以看到，靶机的445端口是开放的，而永恒之蓝利用的就是445端口的SMB服务，操作系统溢出漏洞。

2、使用 MSF 的永恒之蓝漏洞模块

打开MSF：（MSF每次打开都会有一个随机的界面，花里胡哨的）

如果想要看其它界面可以输入 banner 查看

搜索 ms17-010 代码：

一般会有4个结果

这里可以得到一些工具，其中 auxiliary/scanner/smb/smb_ms17_010 是永恒之蓝扫描模块， exploit/windows/smb/ms17_010_eternalblue 是永恒之蓝攻击模块。

一般配合使用，前者先扫描，若显示有漏洞，再进行攻击。

3、使用ms17-010扫描模块，对靶机进行扫描：

使用模块 use 3 ，可以使用 show options 命令查看设置选项，Required的yes是必须要有的，所以要设置 RHOSTS ，也就是目标靶 set rhosts 192.168.0.64 。然后扫描探索一下 exploit 。

如果出现 Host is likely VULNERABLE to MS17-010! 的情况，就说明存在漏洞。

4、使用ms17-010攻击模块，对靶机进行攻击

使用模块 use 0 ，可以使用 show options 命令查看设置选项，Required的yes是必须要有的，所以要设置 RHOSTS ，也就是目标靶 set rhosts 192.168.0.64 。然后执行攻击： exploit 或 run 。

攻击完成！
