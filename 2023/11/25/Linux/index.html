防火墙

常用的Linux防火墙有以下几种：

  1. iptables：iptables是Linux系统中最常用的防火墙工具，它可以过滤网络数据包，从而保护系统安全。
  2. firewalld：firewalld是CentOS 7及以上版本中默认的防火墙工具，它是一个动态防火墙管理器，可以根据应用程序的需要动态地打开和关闭端口。
  3. ufw：ufw是Ubuntu系统中的默认防火墙工具，它基于iptables，提供了一个更简单的用户界面，易于配置和管理。
  4. nftables：nftables是Linux内核中的一个新的网络过滤和分类框架，可以替代iptables。它提供了更快的数据包处理速度和更灵活的规则语法。

Linux常用的防火墙有iptables和firewalld。其中，iptables是比较老的防火墙，而firewalld是新一代的防火墙。在CentOS 7及以上版本中，firewalld已经取代了iptables成为默认的防火墙。

目录详解

/ (根目录): 根目录是整个文件系统的起点，包含了所有其他文件和目录。在 Linux 中，所有的文件和目录都位于根目录下或者是根目录的子目录中。

/bin (二进制)：包含了系统启动和运行所需的基本命令，例如 ls、cp、mv 等。

/boot：包含了系统引导时所需的文件，如内核镜像和引导加载程序配置文件，自己的安装别放这里。

/dev (设备文件)：包含了设备文件，Linux 中一切皆文件，设备也不例外。例如硬盘、键盘等设备都以文件形式存在于这个目录下。

/etc (配置文件)：包含了系统的配置文件，例如网络配置、软件配置等。大部分的系统配置文件都存放在这个目录下。

/home (用户主目录)：每个用户在 Linux 系统中都有一个对应的主目录，通常位于 /home 目录下。例如，用户 “user1” 的主目录就是 /home/user1。

/lib (库文件)：包含了许多程序在运行时所需的共享库文件，为可执行程序提供了必要的函数库支持。其作用类似于windows里的DLL文件。

/lib64：类似于 /lib，不过这里存放的是 64 位系统的库文件。

/media：这是linux2.6内核的一个很大的变化。该目录下安装了2.6内核中新出现的一个文件系统sysfs。centos7迁移到/run/media

/mnt：系统提供该目录是为了让用户临时挂载别的文件系统的，我们可以将外部的存储挂载在/mnt上，然后进入该目录就可以查看里面的内容了。和media差不多。

/opt：这是给主机额外安装软件所摆放的目录。比如你安装一个mysql数据库则就可以放在这个目录下。默认是空的。

/proc (虚拟文件系统)：是一个虚拟文件系统，提供了当前运行进程和内核状态的信息，它是系统内存的映射，我们可以通过直接访问这个目录来获取系统信息。

/root (系统管理员主目录)：只能被 root 用户访问，存放一些系统级别的配置文件、脚本以及其他重要的系统管理工具。

/run：运行时文件系统，存放系统运行时需要变化的文件。

/sbin：包含系统管理员使用的系统管理程序，例如 ifconfig 和 fdisk。

/srv：service缩写，通常被用来存放服务相关的数据，例如 FTP 服务器的文件、WWW 服务器的网页内容等。

/sys：提供了对内核数据结构的访问，允许用户空间程序与内核进行交互。这是linux2.6内核的一个很大的变化。该目录下安装了2.6内核中新出现的一个文件系统sysfs。

/tmp (临时文件)：用于存放临时文件，重启后会被清空。

/usr (用户程序)：包含了大部分用户级别的程序和文件，例如用户安装的软件、游戏、图标等。

/var (变量文件)：包含了经常变化的文件，例如日志文件、临时文件和邮件等。

/lost+found：这个目录一般情况下是空的，当系统非法关机后，这里就存放了一些文件。

/www：可以用于存储Web应用程序或网站文件，/www目录没有特别的用途，它只是一个常规目录，可以根据需要进行使用和管理。

包管理器

在Linux系统中，不同的发行版使用不同的包管理器来更新、下载、安装和卸载软件包。下面是各个发行版常用的命令：

CentOS/RHEL（Red Hat Enterprise Linux）：使用的包管理器是 yum

yum（Yellowdog Updater, Modified）是一个基于 rpm 的软件包管理器，它提供了更高级的功能，如自动解决软件包依赖关系、远程软件包仓库的管理等。

虽然 yum 是首选的软件包管理器，但您仍然可以使用 rpm （Red Hat Package Manager）直接安装软件包。rpm 是一种低级工具，它可以用于安装、查询、验证和卸载软件包。但是，使用 rpm 进行软件安装可能需要手动解决依赖关系，并且不会自动从存储库中下载依赖软件包。

Red Hat及其衍生版（如Fedora、CentOS）：使用 DNF （Dandified YUM）作为包管理器。DNF是对旧版 YUM 的改进，提供了更快的包管理和依赖解决。

  • 更新软件包列表：yum update
  • 显示软件包信息：yum list
  • 下载安装软件包：yum install 软件名称
    • （省略安装：yum install -y 软件名称）
  • 卸载软件包：yum remove 软件名称

Debian及其衍生版（如Ubuntu、Kali Linux）：使用 APT （Advanced Package Tool）作为其主要的包管理器。

除了 APT ，Kali Linux还支持其他包管理工具，如 dpkg 、 apt-get 和 aptitude 等。

在过去，apt-get 是默认的包管理命令，而 apt 是 apt-get 的简化版本。然而，自从APT版本1.1之后，apt命令已经成为新的推荐使用方式，并且提供了更简洁的语法和更友好的输出。

  • 更新软件包列表：apt update 或者 apt-get update

    • 是用于更新 Ubuntu 系统软件包列表的命令。执行命令时，系统会连接到 Ubuntu 软件包仓库，并获取最新的软件包信息。这些信息包括可用的软件包、它们的版本、依赖关系等等。

  • 下载安装软件包：apt install 软件名称 。

    • （省略安装：apt-get install -y 软件名称）

  • 重新安装软件包：apt reinstall 软件名称 或者 apt-get install --reinstall 软件名称

  • 卸载软件包：apt remove 软件名称

    • 卸载指定的软件，但保留配置文件

  • 卸载软件：apt purge 软件名称 或者 apt-get --purge remove 软件名称

    • 卸载指定的软件，同时删除配置文件

  • 删除自动安装的软件包：apt-get autoremove

    • 这些软件包当初是为了满足其他软件包对它的依赖关系而安装的，而现在已经不再需要了

  • 升级软件：apt upgrade

    • 此命令用于从/etc/apt/sources.list中列出的源安装系统上当前安装的所有包的最新版本

需要注意的是，安装和卸载软件包时需要使用管理员权限（通常使用sudo命令）。

net-tools

在Linux系统中，net-tools是一个常用的网络工具包，它提供了一些用于网络配置和监控的命令行工具。安装net-tools可以让你在终端中使用以下常见的网络工具：

  1. ifconfig：用于配置和显示网络接口的信息，包括IP地址、子网掩码、MAC地址等。
  2. netstat：用于显示网络连接、路由表、网络接口统计信息等。
  3. route：用于配置和显示网络路由表。
  4. arp：用于显示和操作系统的ARP缓存，用于解析IP地址和MAC地址之间的映射关系。
  5. iptables：用于配置Linux防火墙规则。

虽然在最新的Linux发行版中，net-tools已经被iproute2工具包取代，iproute2提供了更强大和灵活的网络管理功能。但是，在某些旧版的Linux发行版或特定的网络环境中，你可能仍然需要安装net-tools来使用这些常用的网络工具。

yum install net-tools.x86_64 是指定安装的是net-tools软件包的x86_64架构版本，这是一种特定的硬件架构。这个命令会安装适用于x86_64架构的net-tools软件包。

yum install net-tools 是没有指定具体版本的命令，它会安装适用于系统架构的默认版本的net-tools软件包。这个命令会根据系统的架构自动选择合适的版本进行安装。

软件源

在Linux系统中，要查看已安装的软件源和正在启用的软件源，你可以执行以下操作：

Debian/Ubuntu系统

  • 查看已安装的软件源：cat /etc/apt/sources.list
  • 查看已启用的软件源：grep -v '^#\|^$' /etc/apt/sources.list /etc/apt/sources.list.d/*

CentOS/RHEL系统

  • 查看所有yum软件源：yum repolist all
  • 查看已启用的yum软件源：yum repolist enabled

这些命令将显示已安装的软件源以及当前正在启用的软件源。

安装软件源：yum -y install epel-release //安装 EPEL 软件源
