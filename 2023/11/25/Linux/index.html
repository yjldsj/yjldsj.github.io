    防火墙

    常用的Linux防火墙有以下几种：

    1. iptables：iptables是Linux系统中最常用的防火墙工具，它可以过滤网络数据包，从而保护系统安全。
    2. firewalld：firewalld是CentOS 7及以上版本中默认的防火墙工具，它是一个动态防火墙管理器，可以根据应用程序的需要动态地打开和关闭端口。
    3. ufw：ufw是Ubuntu系统中的默认防火墙工具，它基于iptables，提供了一个更简单的用户界面，易于配置和管理。
    4. nftables：nftables是Linux内核中的一个新的网络过滤和分类框架，可以替代iptables。它提供了更快的数据包处理速度和更灵活的规则语法。

    Linux常用的防火墙有iptables和firewalld。其中，iptables是比较老的防火墙，而firewalld是新一代的防火墙。在CentOS 7及以上版本中，firewalld已经取代了iptables成为默认的防火墙。

    目录

    目录详解

    / (根目录): 根目录是整个文件系统的起点，包含了所有其他文件和目录。在 Linux 中，所有的文件和目录都位于根目录下或者是根目录的子目录中。

    /bin (二进制)：包含了系统启动和运行所需的基本命令，例如 ls、cp、mv 等。

    /boot：包含了系统引导时所需的文件，如内核镜像和引导加载程序配置文件，自己的安装别放这里。

    /dev (设备文件)：包含了设备文件，Linux 中一切皆文件，设备也不例外。例如硬盘、键盘等设备都以文件形式存在于这个目录下。

    /etc (配置文件)：包含了系统的配置文件，例如网络配置、软件配置等。大部分的系统配置文件都存放在这个目录下。

    /home (用户主目录)：每个用户在 Linux 系统中都有一个对应的主目录，通常位于 /home 目录下。例如，用户 “user1” 的主目录就是 /home/user1。

    /lib (库文件)：包含了许多程序在运行时所需的共享库文件，为可执行程序提供了必要的函数库支持。其作用类似于windows里的DLL文件。

    /lib64：类似于 /lib，不过这里存放的是 64 位系统的库文件。

    /media：这是linux2.6内核的一个很大的变化。该目录下安装了2.6内核中新出现的一个文件系统sysfs。centos7迁移到/run/media

    /mnt：系统提供该目录是为了让用户临时挂载别的文件系统的，我们可以将外部的存储挂载在/mnt上，然后进入该目录就可以查看里面的内容了。和media差不多。

    /opt：这是给主机额外安装软件所摆放的目录。比如你安装一个mysql数据库则就可以放在这个目录下。默认是空的。

    /proc (虚拟文件系统)：是一个虚拟文件系统，提供了当前运行进程和内核状态的信息，它是系统内存的映射，我们可以通过直接访问这个目录来获取系统信息。

    /root (系统管理员主目录)：只能被 root 用户访问，存放一些系统级别的配置文件、脚本以及其他重要的系统管理工具。

    /run：运行时文件系统，存放系统运行时需要变化的文件。

    /sbin：包含系统管理员使用的系统管理程序，例如 ifconfig 和 fdisk。

    /srv：service缩写，通常被用来存放服务相关的数据，例如 FTP 服务器的文件、WWW 服务器的网页内容等。

    /sys：提供了对内核数据结构的访问，允许用户空间程序与内核进行交互。这是linux2.6内核的一个很大的变化。该目录下安装了2.6内核中新出现的一个文件系统sysfs。

    /tmp (临时文件)：用于存放临时文件，重启后会被清空。

    /usr (用户程序)：包含了大部分用户级别的程序和文件，例如用户安装的软件、游戏、图标等。

    /var (变量文件)：包含了经常变化的文件，例如日志文件、临时文件和邮件等。

    /lost+found：这个目录一般情况下是空的，当系统非法关机后，这里就存放了一些文件。

    /www：可以用于存储Web应用程序或网站文件，/www目录没有特别的用途，它只是一个常规目录，可以根据需要进行使用和管理。

    在Linux上安装redis
    1. 在官网下载tar.gz的安装包然后上传，或者通过wget的方式下载　【wget http://download.redis.io/releases/redis-4.0.1.tar.gz】
    2. 解压【tar -zxvf redis-4.0.1.tar.gz】
    3. redis是C语言开发，安装redis需要先将官网下载的源码进行编译，编译依赖gcc环境。
    * 查看gcc版本【gcc -v】
    * 若没有安装则，使用命令【yum install gcc】或者【yum install -y gcc】或者【yum install gcc-c++】进行安装；
    ? 更新【yum update -y】
    3. 编译，通过make来编译，make是自动编译，会根据Makefile中描述的内容来进行编译。
    * 【cd /usr/local/redis/redis-6.2.6】
    * 【make】
    * 【make install】实际上，就是默认安装在 /usr/local/bin/目录，这个目录默认就在系统环境变量中。
    * 需要更改可以在后面接【make install PREFIX =/usr/local/redis】后面的是你想要安装的路径
    - make install安装完成后，会在/usr/local/bin目录下生成下面几个可执行文件，它们的作用分别是：
    - redis-server：Redis服务器端启动程序
    - redis-cli：Redis客户端操作工具。也可以用telnet根据其纯文本协议来操作
    - redis-benchmark：Redis性能测试工具
    - redis-check-aof：数据修复工具
    - redis-check-dump：检查导出工具
    4. 从 redis 的源码目录中复制 redis.conf 到 redis 的安装目录
    * 【cp /usr/local/redis/redis-6.2.5/redis.conf /usr/local/redis/bin/】
    5. 修改 redis.conf 文件，【vi redis.conf】
    * 把【daemonize no】改为【daemonize yes】
    * 关闭【protected-mode】模式，此时外部网络可以直接访问，设置成【protected-mode no】
    * 备注：如果想要设置指定IP连接redis，只需要修改redis.conf文件中bind配置项即可。如果不限IP，将127.0.0.1修改成0.0.0.0即可。
    * port 端口号
    * 【requirepass foobared】 -> 设置密码
    6. 服务启动与关闭
    * 【ps -ef | grep redis】查看Redis是否正在运行
    * 【./redis-server redis.conf】或者【./redis-server】前台启动，按“Ctrl + C”退出
    * 【./redis-cli】开启客户端服务；退出【quit】
    * 【./redis-cli shutdown】关闭服务
    7. 设置开机自动启动
    1. 切换到/lib/systemd/system/目录，创建redis.service文件。命令【cd /lib/systemd/system/】【vim redis.service】
    2. 文件内容如下：
    * [Unit]
    * Description=redis-server
    * After=network.target
    *
    * [Service]
    * Type=forking
    * # ExecStart需要按照实际情况修改成自己的地址
    * ExecStart=/usr/local/redis/bin/redis-server /usr/local/redis/bin/redis.conf
    * PrivateTmp=true
    *
    * [Install]
    * WantedBy=multi-user.target
    8. 相关redis服务
    * 【systemctl status redis.service】 #查看服务当前状态
    * 【systemctl start redis.service】 #启动redis服务
    * 【systemctl stop redis.service】 #停止redis服务
    * 【systemctl restart redis.service】 #重新启动服务
    * 【systemctl enable redis.service】 #设置开机自启动
    * 【systemctl disable redis.service】 #停止开机自启动
    9. 如果主机连不上redis可以检查下防火墙或者开放redis端口
    1. 关闭防火墙
    * # 检查防火墙状态 看到active(running)就意味着防火墙打开了

    * # 关闭防火墙
    * 【sudo systemctl stop firewalld】
    * # 开启防火墙
    * 【sudo systemctl start firewalld】

    * 【sudo systemctl disable firewalld】
    2. 开放端口
    * # 开放redis端口
    * 【firewall-cmd –zone=public –add-port=6379/tcp –permanent】
    * # 应用
    * 【firewall-cmd –reload】
    10. 卸载redis
    * 卸载redis服务，只需把/usr/local/bin/目录下的redis删除即可
