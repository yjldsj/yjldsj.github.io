github+hexo搭建博客

介绍

Hexo是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。即把用户的markdown文件，按照指定的主题解析成静态网页。

这次搭建博客是在windows的环境下进行的。

准备前提

1、已安装 Node.js 。

因为Hexo是基于nodeJS编写的，所以需要安装一下nodeJs和里面的npm工具。

2、已安装 Git 。

3、已安装 Typora 。

写博客用的。

4、注册好了一个github账号

5、要有代理软件，也就是梯子。

如果没有安装 Node.js ，请看下面的安装步骤。

去 node.js 或者 Node.js 中文网 里面下载，选择 LST版本的进行安装。

Current 指的是当前发布的最新node版本，里面包含了最新的功能特性，但会不稳定，会不定期的进行更新优化或者修复问题。

LTS指的是长期支持的node版本，即稳定版，里面包含的功能就是稳定的。

运行 corepack enable 启用 Corepack。

启用后自动安装 Yarn 和 PNPM，这样以后的命令都可以换成喜欢的包管理器

全部下一步，唯一要注意的就是安装位置。

安装完成后，输入命令 node -v ，命令 npm -v 显示对应的版本，说明安装成功。

只是搭建博客的话，对于 环境配置 无所谓，如果要进行其它的操作，可以去百度一下 环境配置 。

如果没有安装 Git ，请看下面的安装步骤。

去 Git - Downloading Package 上进行下载，选择 64-bit Git for Windows Setup 进行下载安装。

全部下一步，唯一要注意的就是安装位置。

安装完成后，在cmd或者 Git Bash 中，输入 git 命令，显示了对应结果，说明 git 可以工作了。

Hexo 有多种图片插入方式，可以将图片存放在本地引用或者将图片放在 CDN 上引用。

插入图片-本地引用

1、本地引用–绝对路径

当 Hexo 项目中只用到少量图片时，可以将图片统一放在网站目录下的 source/images 文件夹中，通过 markdown 语法访问它们。

这里的 / 指的是根目录，对于hexo，资源文件的根目录就是source。

图片既可以在首页内容中访问到，也可以在文章正文中访问到。

2、本地引用–相对路径 （推荐）

图片除了可以放在统一的 images 文件夹中，还可以放在文章自己的目录中。文章的目录可以通过配置 _config.yml 来生成。

打开项目根目录中的 _config.yml 文件，将 _config.yml文件中的配置项进行配置

然后执行命令 hexo new "post_name"，在 source/posts 中会生成文章 post_name.md 和同名文件夹 post_name
